<div class="section-label"><span class="section-num">[01]</span> &nbsp; About</div>

My day-to-day is reviewing how products handle authentication, authorization, and the trust
between services. The vulnerabilities I end up reporting usually start as small inconsistencies
&mdash; a stale session, a permission check on the wrong layer, state that wasn't supposed to leak.

Tools help cover ground, but findings come from careful reading. I want to know what an
application *expects* to be true, then test what happens when those expectations break.
That's where the interesting bugs hide, and it's why I treat scanner output as a starting
point, not a finish line.

Outside of live programs I publish writeups, contribute to security tooling, and keep up with
the platforms I follow. Bug classes shift over time, so staying current is part of the work.

<div class="section-label"><span class="section-num">[02]</span> &nbsp; Focus</div>

- **Bug Bounty Hunting** &mdash; Finding and responsibly disclosing vulnerabilities through platforms like HackerOne and YesWeHack
- **Security Research** &mdash; Exploring web application vulnerabilities, API security flaws, and authentication mechanisms
- **Technical Documentation** &mdash; Writing detailed vulnerability reports and security advisories
- **Continuous Learning** &mdash; Participating in CTF competitions and security training platforms
